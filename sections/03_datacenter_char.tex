\begin{frame}{Раздел 3: Характеристика рабочей нагрузки разработки LLM}
    \begin{itemize}
        \item \textbf{Цель:} Сравнить LLM с предыдущими DL нагрузками и выявить особенности использования ресурсов и сбоев.
        \item \textbf{Методология:} Анализ 6-месячных логов (март-август 2023) из датацентров Seren и Kalos (всего 4704 GPU A100).
        \item \textbf{Типы работ:} Претренинг (Pretraining), дообучение (Fine-tuning), выравнивание (Alignment), оценка (Evaluation), подготовка данных (Data preparation).
    \end{itemize}
\end{frame}

\begin{frame}{3.2 Характеристики рабочей нагрузки}
    \begin{itemize}
        \item \textbf{Длительность работ:} В среднем 2.7--12.8x \textbf{короче}, чем в предыдущих DL трейсах (из-за большого числа коротких задач оценки).
        \item \textbf{Задержка в очереди:}
            \begin{itemize}
                \item Вопреки прошлым выводам, \textbf{крупные} задачи (претренинг) имеют \textbf{меньшую} задержку (медиана 28 сек в Kalos).
                \item \textbf{Короткие} задачи оценки ждут дольше всех (медиана 7.8 мин в Kalos), так как используют только остаточные ресурсы.
            \end{itemize}
        \item \textbf{Распределение потребления ресурсов (Kalos):}
            \begin{itemize}
                \item Претренинг: 3.2\% работ, но \textbf{94.0\% GPU-времени}.
                \item Оценка: 92.9\% работ, но лишь \textbf{0.8\% GPU-времени}.
            \end{itemize}
    \end{itemize}
\end{frame}

\begin{frame}{3.3 Структура использования ресурсов}
    \begin{itemize}
        \item \textbf{Высокая утилизация GPU:}
            \begin{itemize}
                \item Память GPU (медиана 75\%, 60 ГБ) и вычислительная способность (медиана 99\% в Kalos) загружены сильно.
                \item Это намного выше, чем в старых DL трейсах (PAI: 10\% и 4\%).
                \item Вывод: LLM интенсивны к вычислениям и памяти; методы разделения GPU (\textit{GPU sharing}) неэффективны.
            \end{itemize}
        \item \textbf{Недоиспользование сопутствующих ресурсов:}
            \begin{itemize}
                \item CPU, хост-память и сеть загружены слабо (CPU в Kalos в среднем на 20\%).
                \item Основной бутылочное горлышко — операции ввода-вывода при подготовке данных.
            \end{itemize}
    \end{itemize}
\end{frame}

\begin{frame}{3.3 Длительный простой GPU при оценке}
    \begin{itemize}
        \item Профилирование задачи \texttt{HumanEval} показало:
            \begin{itemize}
                \item \textbf{29.5\% времени:} Загрузка модели и предобработка данных (CPU/диск).
                \item \textbf{19.0\% времени:} Тестирование синтезированных программ (CPU).
                \item \textbf{Только 51.5\% времени:} Непосредственно инференс на GPU.
            \end{itemize}
        \item \textbf{Проблема:} GPU простаивает почти половину времени выполнения задачи оценки, что снижает эффективность и увеличивает задержки в очереди.
    \end{itemize}
\end{frame}

\begin{frame}{3.4 Анализ сбоев}
    \begin{itemize}
        \item \textbf{Типы ошибок:}
            \begin{itemize}
                \item \textbf{На старте (быстрое завершение):} Ошибки конфигурации, некорректные пути к данным (в основном для коротких задач).
                \item \textbf{Во время выполнения:} Сбои оборудования (GPU, InfiniBand, диск), характерные для долгих задач претренинга.
            \end{itemize}
        \item \textbf{Влияние:} Сбои инфраструктуры сильно снижают эффективность претренинга, требуя быстрой диагностики и восстановления.
    \end{itemize}
\end{frame}
